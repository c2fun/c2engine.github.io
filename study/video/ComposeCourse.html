概述

护送任务，是保护NPC从一个地方到另一个地方的任务形式，完成任务的条件是：玩家需要全程在npc周围，确保npc没有死亡，安全到达目的地。

合成流程

这里讲述一个标准护送任务的合成的流程：
1、打开任务编辑器，添加一个新任务。在获得任务的模块填写护送任务执行脚本-21，并把接任务npc隐藏20秒左右；在完成任务检测条件模块填写完成事件次数的条件。导出任务脚本
2、打开关卡编辑器，摆放接任务、交任务以及被护送的npc。接任务、交任务npc按任务文档的描写摆放；被护送npc放在“npc集中营”里，被护送的npc拥有自己的怪物id。tb_npc导入数据库
3、在关卡编辑器里取被护送npc行走路径（坐标），配置到tb_npcaction。tb_npcaction导入数据库
4、给被护送npc配置AI，到达目的地，使玩家完成这个护送任务，并移除自己。TB_MOBINFOBASE、TB_MOBINFOEXT、TB_AI_INFO、TB_AI_CONFIG导入数据库
5、更新重启SA，就可以测试了

任务合成

执行手工脚本

在获得任务的同时，执行护送任务脚本：
-21 参数1：NPC DBID （被护送npc的id）参数2：临时阵营ID（-1表示不使用临时阵营）
说明：
1、参数1：填写被护送npc的DBID。一般的护送任务，接任务npc的名字和被护送npc的名字是一样的，但是他们的DBID不一样，注意填写正确。
2、参数2：临时阵营，这里一般填-1。

检测完成条件

完成事件次数，参数1：1（填1表示是护送任务的检测） 参数2：NPC DBID（被护送npc的id）参数3：护送次数
说明：
1、参数1：护送任务填1，表示事件类型是护送npc。该值填2的时候，是另一种任务形式，原地保护npc；填写其他值，暂时没有意义
2、参数2：填写被护送npc的DBID。这里填写的ID和护送任务执行脚本填写的保持一致
3、参数3：完成事件的次数。护送这个npc多少次。比如押镖，押三次才能完成任务

关卡合成

npc摆放

接任务npc摆放：
在什么地方接任务，就摆在什么地方，接任务的地点，即护送开始的地方
交任务npc摆放：
在什么地方交任务，就摆在什么地方，交任务的地点，不一定是护送结束的地方
被护送npc摆放：
被护送的npc一般放在“npc集中营”里。
当执行护送任务的时候，这个npc才会出现。
例如：在杰克·奥尼尔那里接到一个任务，把他护送到空军基地，再接任务的时候，我们需要把“接任务的杰克·奥尼尔”隐藏，同时用-21脚本召唤出“被护送的杰克·奥尼尔”，我们一路上护送的，是一个“克隆人”。

tb_npcaction配置

被护送的npc，要在tb_npcaction表里，配置他一路上的行为，主要是被护送npc的行走路线，也可以有休息、说话、打怪等行为
具体配置方法请参见：

AI合成

遭遇抢劫的AI

在护送的途中，突然出现一批怪物，堵截被护送的npc。
·事件：到达目标（事件id 13）
AI所有者，被护送的npc
·条件：在某坐标点附近（条件id 14）
参数1：x坐标
参数2：z坐标（高度y坐标不用填）
检测目标：被护送的npc
·处理事件：召唤npc（事件id 10）
参数1：配置id
参数2：不需要填写
处理目标：被护送的npc

这个AI的含义是：被护送的npc，走到某个坐标点的时候，他就会触发一个召唤事件，召唤出怪物来。
召唤事件的配置文件路径：\tw-server\serverarea\script\server\config\ai

   _tSummonEx = {
                 [1] ={                                  --召唤事件id
                 delay = 2000;                           --延时时间（ms）
                   {dbid=1166, radius=4.0, count=2, camp=3},--dbid，召唤的npc id；radius，召唤npc的范围（m），以召唤者为圆心；count 数量；camp 阵营
                   };
                 }

护送完成的AI

到达目的地，被护送的npc，使玩家完成任务，并同时删除掉自己。
·事件：到达目标（事件id 13）
事件所有者：被护送的npc
·条件：在某坐标点附近（条件id 14）
参数1：x坐标
参数2：z坐标
这里填的坐标就是被护送npc的终点了，触发这个AI的时候，npc会在这个点上消失
·处理事件：移除自己、完成护送（事件id 8）
处理目标：被护送的npc
处理目标的目标：被护送的npc

这个AI的含义是：
被护送的npc，到达终点坐标，触发完成护送的事件，使玩家完成任务，并移除自己

常见问题

·在执行-21护送任务脚本的同时，最好把接任务npc隐藏掉。一是避免两个一摸一样npc同时出现的尴尬，二是避免玩家反复接取任务
·在配置被护送npc的行走路线时，路径点尽量多一些，密一些，以避免发生npc转圈、停步、绕远等情况
·tb_npcaction表里，npc的终点，和完成护送的AI配置里的坐标点，保持一致，以免发生不能完成任务的情况
·被护送npc的npc类型，最好填211，即怪物类型，以避免在护送的过程中，玩家和被护送npc频繁对话，而引起不必要的麻烦。npc在执行npcaction设定的路线时，如果有玩家前来对话，npc会停下来
·护送任务以及保护任务，任务失败之后，不用放弃任务也可以重新接到任务

应用举例

被抢劫的商人

这类的护送任务，在护送的途中，会出现一群怪物，袭击被护送的npc。通过召唤AI来实现。

穿越怪物地带

最简单的护送任务，被护送npc的行走路线要通过一个怪物分布区，利用地图上已有的怪物来增加难度。

多次护送

完成任务的条件是，护送npc多次。比如押镖，要成功押镖三次才能完成任务。这样的任务，反复做多次，为避免护送过程的枯燥，可以通过配置AI发生随机事件。比如，镖车走到某个坐标点的时候，有几率对自己使用一个秒杀技能，镖车毁坏了；镖车走到一些坐标点的时候，随机召唤出山寨大王、巡山队长、喽啰等几种怪，山寨大王出现的几率最低，但是难度大，掉落好。

节外生枝

不仅仅是护送，还需要跟着npc完成其他的任务。在敌占区发现了一个被俘的士兵，这个士兵是秘密潜进来执行特殊任务的，不幸被捕。这个士兵的任务是杀掉敌军小队长以及破坏敌占区的三个弹药库。护送开始，先跟着士兵找到敌军小队长，杀掉小队长再跟着npc破坏三个弹药库，然后才走上出去的路线。

新增配置

配置文件

D:\development\twilight\tw-server\serverarea\script\server\config\ai\ai_config_s.lua

配置

EscortConfig = { 

   MaxShareDistanceSQ = 100; -- 护送任务最大共享范围的平方(米).
   MaxShareTime = 5000; -- 护送最大共享时间(毫秒).
   MaxOutRangeTime = 10000; -- 护送超出距离最大时间(毫秒).
   MaxEscortDistanceSQ = 400; -- 护送最大距离平方(米).

}
